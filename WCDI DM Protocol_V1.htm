1. Purpose

The purpose of this Data Management Protocol is to set general data management practices within the Wageningen Centre for Development Innovation (WCDI). More specifically, these data management practices aim to prevent data loss and increase reusability of data by complying with the Dutch Association of Universities (VSNU) Code of Conduct and WUR data policy. The WUR data policy is guided by, amongst others, the FAIR principles (Findable, Accessible, Interoperable, Reusable) and the motto to share data ‘as open as possible, as closed as necessary’. The WUR data policy requires:

·        Each university department and research institute under WUR to write and update a Data Management Protocol (group level).

·        Project leads to having a Data Management Checklist / Plan (project-level).

·        Secure, safe and shareable storage of (research) data during research and project implementation.

·        Archiving of data sets underlying publications (articles/reports/knowledge products/course materials) for at least 10 years after research.

·        Registration of archived data sets in appropriate locations (e.g., in Pure for publications).[1]

While the WUR data policy applies for research data only, the data management practices outlined in this Data Management Protocol for WCDI are adapted to apply to all data produced within (research and non-research) projects and courses implemented at WCDI.

2. Duration and review procedure for this protocol

The protocol is binding for projects from 1 February 2023. The protocol will be reviewed by the WCDI Data Stewards team annually in consultation with the Knowledge Manager and Business Unit Manager Subsequent version numbers will be assigned to this protocol and old versions archived in the W:-drive.

4. Data ownership, sharing and accessibility

WUR’s data ownership policy stipulates that WUR is the owner of data and entitled to data(bases) created by any WUR staff member within the scope of their employment. This means that WUR researchers, students and users of its facilities are not the owner nor right holders of the data(base).

WUR ownership should be discussed in the case the funder or lead organization of the project imposes ownership conditions. In other words, the party that has ownership over the data decides what others are allowed to do with it; how the data are shared and accessed. If you collect data from/with/on behalf of an external party, you may have other agreements with that party, and these generally overrule WUR’s data ownership policy.

The WCDI data sharing guidelines [add link] provide more information on sharing (research) data – including the preparation of a Data Sharing Agreement – with third parties and for working with data when executing projects with multiple partners. In the latter case, it is highly recommended to prepare such an Agreement if a clause on data sharing in the project contract is lacking, no matter if the data is owned by WUR, an external partner, or if ownership is shared.

5. Safe and shareable storage of data during a project or course

The WUR data policy requires secure and shared storage of data. The classification of the data determines how secure the storage must be. For high-risk data, increased security measures are needed. Several secure and shared storage solutions are provided by WUR. Using these storage solutions ensures compliancy with the WUR data policy for the data collected at any project or course within WCDI. As such, data are stored in locations that are backed up automatically/synchronized (so NOT on hard disks, USBs, laptops, personal DropBox folders). This prevents data loss and ensures that data can be accessed by authorised parties in case of emergency or departure from WUR.

Guidance on what type of data should be stored where will be specified in the Data Management Checklist / Plan in accordance to the storage solutions provided by WUR. It is important to ensure that data safety, integrity, and security still comply with WUR standards.

Upon termination of a WUR account, all files on the M:-drive and/or OneDrive for Business are automatically deleted, including permanent deletion of data stored there. Therefore, WUR policy on data storage stipulates that all data in a personal folder/personal cloud storage should be stored in a department or project folder on the W:-drive. Therefore, all data need to be backed up to the W:-drive on a regular basis. Back-up frequency will be determined by the Project Leaders / Course Coordinators / Secretariat / Project Support Group. In case of MS Teams, every Team must have at least 3 owners. This is to guard against an employee leaving WCDI and MS Teams access gets blocked.

Data Storage locations

MS Teams

 

This is the primary platform for all collaborations. All WCDI staff have access to the latest version of this platform, which operates seamlessly as a Windows application and web-based tool. Data stored in MS Teams (in the “files” tab) gets automatically stored in One Drive. There is a “sync” function where a virtual drive is created in the local hard drive and files can be accesses/edited just like local files.

OneDrive

 

This is the cloud storage platform of Microsoft and files can be stored locally, while they also sync to the cloud. Files stored in this manner (without linking to MS teams or SharePoint) are only accessible to the creator. However, the creator can give access to any person having an Office 365 Business account. These files are permanently deleted upon termination of a WUR Account (see above).

SharePoint

 

The older version of MS Teams, having similar functions, but more powerful on the web-based platform. The ‘files’ part of MS Teams is a user-friendly ‘shell’ for storing files on Sharepoint.

Local drive (M:-drive)

 

This is the physical hard drive of the staff’s laptop computer where the Windows “Document” folder is mapped. This location is private to the user and even if the user saves data in this place, temporarily, they need to be moved to the appropriate MS Teams locations (that have shared ownership) and W:-drive for proper team accessibility.

W:-drive

The W:-drive is an ideal location to back up data from personal folders (M:-drive). This guarantees continuity of access to data if the personal folder or are deleted (due to change of job, etc.), complying with “WUR policy for storage of (research) data”.

Related to project management (Uniform Project Dossier – UPD)

Uniform Project Dossier (UPD) is storage of project (management) data in Sharepoint. Within WUR an environment within Sharepoint is realised in which one project file is automatically created for each new project with access to all parties involved in the preparation, execution and support of projects[2].

Related to research data

Other storage locations for research data specifically include Pure and Yoda

- Pure is the source system for WUR research output from articles and books to datasets and press clippings. Data is entered and stored in Pure by your research unit and made publicly available. The data is also used for publication lists on websites. See the following link for more details on Pure and procedures to register information in the system: (https://intranet.wur.nl/umbraco/en/practical-information/research-support/writing-publishing/)

- Yoda is a research data management service that enables you and your research partners to securely deposit, share, publish and preserve large amounts of research data during all stages of a research project. See the following link for more information: https://www.wur.nl/en/value-creation-cooperation/collaborating-with-wur-1/wdcc/research-data-management-wdcc/doing/yoda.htm

 

 

 

6. Data archiving and registration after a project or course ends

What data needs to be archived and where?

In compliance with the WUR data policy, all WCDI data produced within projects and courses need to be archived to the W:-drive to enable verification and reusability of the data.

In which format should the data be stored?

Data can be stored in any format suitable for analysing and accessing the data. However, as much as possible, data files should also be stored in open file formats (e.g. .txt, .csv, etc.) to facilitate accessibility and reusability.

Registration of research data sets

Registration of research data underlying any publication or other knowledge product (archived on the W-drive) are required to be registered in Pure, by sending the persistent identifier or pathway and accompanying publication(s) to data@wur.nl. publications are not required to be registered, but it is recommended.

 

Restricted or closed data

When access to data needs to be restricted or closed, the data still needs to be safely archived within WUR. This means that the data should be archived in a system that fits the classification (intranet WUR, login required) of the data and where access to the data is managed (e.g. restricted to certain persons).

Who has access to the data archived at W-Drive?

Data on the:

·          W-drive can only be changed or removed by Portfolio Support Group / Secretariat.

·          Secretariat authority to grant read/write access to W-drive to WCDI staff.

 

7. Data documentation and metadata

All complex data sets should have a “readme.txt” file which will have information on the contents of that folder to make datasets findable. This enables easy searching for contents.[3]  For more information, refer to the link (https://bit.ly/3S3UBr2).

8. Addressing complaints, conflicts about data ownership/management

All conflicts and complaints will be first addressed with the Knowledge Manager. Subsequent escalations will be made in the following sequence.

1.    Complainant

2.    Knowledge Manager

3.    Business Unit Manager

The Business Unit Manager will have the final authority over any standing decisions regarding the abovesaid conflicts. This applies only to cases where WUR has the ownership of the data.

Data management will lie with the Project Leader / Course Coordinator (for projects and courses) and the Secretariat / Portfolio Support Group for administrative data. Refer to the “roles and responsibilities” section of this document for clarity. Conflicts, however, will be addressed as explained in this section.
